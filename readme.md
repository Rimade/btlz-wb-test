# Сервис тарифов WB и выгрузки в Google Таблицы

Сервис выполняет две задачи по ТЗ:

1. **Регулярное получение тарифов коробов WB** — раз в час запрашивает API Wildberries и сохраняет данные в PostgreSQL на каждый день; при повторном запросе за тот же день данные обновляются (upsert по складу и дате).
2. **Регулярное обновление Google Таблиц** — раз в 30 минут выгружает актуальные тарифы из БД в произвольное количество таблиц по их идентификатору. Лист для тарифов: **`stocks_coefs`**. Данные отсортированы по коэффициенту (по возрастанию). Если листа `stocks_coefs` в таблице нет, он создаётся автоматически.

Запуск приложения — одной командой `docker compose up`, без дополнительных манипуляций (миграции выполняются при старте).

## Что нужно для запуска

- Docker и Docker Compose
- Токен WB API (выдаётся после принятия тестового на HH)
- Google Cloud: сервисный аккаунт с включённым Google Sheets API (email + private key в `.env`)

## Конфигурация (без чувствительных данных в репозитории)

В репозитории есть заготовка **`example.env`** — в ней отсутствуют секреты. Параметры БД по ТЗ: пользователь, пароль и база — **`postgres`** (как указано в задании).

## Установка и запуск

1. Клонируй репозиторий и перейди в каталог проекта.

2. Создай `.env` из шаблона и заполни секреты:
   ```bash
   cp example.env .env
   ```
   В `.env` обязательно указать:
   - `WB_API_TOKEN` — токен WB API
   - `GOOGLE_SERVICE_ACCOUNT_EMAIL` — email сервисного аккаунта
   - `GOOGLE_PRIVATE_KEY` — приватный ключ (можно в одну строку с `\n` вместо переносов)  
   Пользователь, пароль и имя БД по умолчанию уже заданы в `example.env` как `postgres`.

3. Запуск одной командой (дополнительные действия не требуются):
   ```bash
   docker compose up
   ```
   Поднимутся контейнеры PostgreSQL и приложения. Приложение при старте выполнит миграции БД и сразу один раз запросит тарифы WB и обновит Google Таблицы. Далее: WB — каждый час, выгрузка в таблицы — каждые 30 минут.

## Как добавить Google Таблицы

- Создай таблицу в Google, создай лист с именем **`stocks_coefs`**.
- Дай доступ на редактирование таблицы почте сервисного аккаунта (`GOOGLE_SERVICE_ACCOUNT_EMAIL`).
- ID таблицы возьми из URL: `https://docs.google.com/spreadsheets/d/<SPREADSHEET_ID>/edit`.
- Добавь этот ID в БД: либо выполни сиды после первого запуска (`npm run knex:dev seed run` локально с подключением к той же БД), либо вставь строку в таблицу `spreadsheets`:
  ```sql
  INSERT INTO spreadsheets (spreadsheet_id, title) VALUES ('твой_spreadsheet_id', 'Название');
  ```
  В сидах уже есть 2–3 заготовки с placeholder ID — их можно заменить на свои.

## Проверка функционирования

1. **Health-эндпоинт (способ проверки работы приложения)**  
   После запуска `docker compose up` приложение поднимает HTTP-сервер на порту из `APP_PORT` (по умолчанию 5000). Проверка доступности и подключения к БД:
   ```bash
   curl http://localhost:5000/health
   ```
   Ожидаемый ответ при рабочем состоянии: `ok` (код 200). При недоступности БД: `unhealthy` (503).

2. **Логи приложения**  
   В логах контейнера `app` должны появляться сообщения:
   - `DB connected`
   - `Migrations up to date`
   - `Health server listening on port 5000`
   - `WB: done, upserted N rows for YYYY-MM-DD`
   - `Sheets: done, ok=... err=...`

3. **PostgreSQL**  
   Подключись к БД (например: `psql -h localhost -U postgres -d postgres`, пароль из `.env`) и проверь данные:
   ```sql
   SELECT * FROM wb_tariffs ORDER BY dt DESC, warehouse_name LIMIT 20;
   ```

4. **Google Таблицы**  
   Открой таблицу, для которой добавлен ID в `spreadsheets`. На листе `stocks_coefs` должны быть заголовки и строки тарифов, отсортированные по коэффициенту по возрастанию.

## Локальная разработка (без Docker)

- Запусти только Postgres: `docker compose up -d postgres`
- Скопируй `.env`, в нём укажи `POSTGRES_HOST=localhost` (и порт при необходимости).
- Миграции: `npm run knex:dev migrate latest`
- Сиды (опционально): `npm run knex:dev seed run`
- Запуск приложения: `npm run dev`

## Структура проекта

- **Точка входа:** `src/app.ts` — подключение к БД, миграции, запуск health-сервера и планировщика.
- **Сервисы:** `src/services/wb.ts` (API WB «Тарифы коробов» и upsert в `wb_tariffs`), `src/services/sheets.ts` (запись в Google Таблицы, создание листа `stocks_coefs` при отсутствии).
- **Планировщик:** `src/scheduler.ts` — node-cron: WB по расписанию `0 * * * *` (каждый час), выгрузка в таблицы `*/30 * * * *` (каждые 30 минут); при старте оба задания выполняются один раз сразу.
- **Health:** `src/server/health.ts` — HTTP GET `/health` для проверки работоспособности и Docker healthcheck.

## Локальная разработка и CI

- **Тесты:** `npm test`
